### 1.4. Уровень жизни и ежедневные расходы

Можно привязать к семье а не только к персонажу

Игрок задаёт **уровень жизни** (§2.7). Система списывает долю от **среднемесячного дохода за последние 30 дней** (`monthly_income`).

```
daily_living_cost = monthly_income × living_percent / 30
```

Если `monthly_income = 0` (безработный смерд), применяется **базовая стоимость выживания** — фиксированная ставка по уровню жизни (см. таблицу ниже).

| Уровень    | living_percent | Базовая ставка (при доходе 0) | Бонусы (модификаторы/день)                       | Штрафы (модификаторы/день)                       |
| ---------- | -------------- | ----------------------------- | ------------------------------------------------ | ------------------------------------------------ |
| Нищета     | 10%            | 0.5 деньги                    | —                                                | здоровье −0.3, настроение −0.2, общая реп. −0.05 |
| Скромность | 25%            | 1.2 деньги                    | —                                                | —                                                |
| Достаток   | 45%            | 2.5 деньги                    | здоровье +0.1, настроение +0.1, общая реп. +0.02 | —                                                |
| Роскошь    | 70%            | 5.0 деньги                    | влияние/связи +0.15                              | накопление замедлено (см. §1.5)                  |
| Мотовство  | 95%            | 8.0 деньги                    | кратковременно общая реп. +0.1 (первые 14 дней)  | риск долгов, метка «расточитель»                 |

Можно регулировать уровнь, но не более доступного уровня

**Согласование со стартом смерда:** при капитале 20 денег и уровне «Скромность» без дохода расход **1.2 деньги/день** → запас **~16 дней** до нуля. Это задаёт давление найти работу в первые 2 недели.

При типичном доходе смерда **40 денег/месяц** (физический труд) и «Скромности»:
`daily_living_cost = 40 × 0.25 / 30 ≈ 0.33 деньги/день` — игрок копит, но медленно.

### 1.5. Эффект роскоши на накопление

При уровне «Роскошь» и «Мотовство» сверх `daily_living_cost` раз в 7 дней срабатывает событие **импульсных трат**:

```
impulse_spend = monthly_income × 0.05 × (living_percent / 0.25)
```

(для Роскоши: ×0.05; для Мотовства: ×0.12)

Если слишком бедный, то копить сложнее из низкого коэф, а при высоком проще, но есть риск непреднамеренных расходов

## 2. Качество товаров (шкала Q)

Единая шкала **Q = 0..100** для сырья, полуфабрикатов и готовой продукции.

### 2.1. Уровни качества

| Уровень    | Диапазон Q | Множитель закупки | Множитель продажи | Δ сословная реп. (купцы) за сделку |
| ---------- | ---------- | ----------------- | ----------------- | ---------------------------------- |
| Халтура    | 0–19       | ×0.6              | ×0.5              | −3                                 |
| Низкое     | 20–39      | ×0.8              | ×0.75             | −1                                 |
| Обычное    | 40–59      | ×1.0              | ×1.0              | 0                                  |
| Хорошее    | 60–79      | ×1.35             | ×1.25             | +1                                 |
| Отличное   | 80–94      | ×1.7              | ×1.5              | +2                                 |
| Мастерское | 95–100     | ×2.2              | ×1.85             | +4                                 |

Функция множителя по непрерывному Q (для промежуточных значений):

```
quality_price_mult(Q) = 0.5 + (Q / 100) × 1.7        // закупка
quality_sell_mult(Q)  = 0.5 + (Q / 100) × 1.35       // продажа
```

### 2.2. Итоговое качество продукта

При нескольких входах (сырьё + навык + мастерская):

```
Q_out = min(Q_inputs) × 0.4 + skill_craft × 0.35 + workshop_bonus × 0.25
```

- `Q_inputs` — качество всех входящих ресурсов (берётся минимум — «слабое звено»).
- `skill_craft` — навык ремесла персонажа или старшего мастера, 0..100.
- `workshop_bonus` — бонус уровня мастерской (см. §4), 0..22.

Результат округляется до целого, ограничивается 0..100.

### 2.3. Скандал (обман покупателя)

Условие: игрок выставляет цену как для Q ≥ 60, а фактический `Q_out < 25`.

| Последствие                     | Значение    |
| ------------------------------- | ----------- |
| Δ сословная репутация (купцы)   | −8          |
| Δ общая репутация               | −5          |
| Шанс потери постоянного клиента | 30%         |
| Повтор в течение 90 дней        | штрафы ×1.5 |

## 3. Поставщики сырья

### 3.1. Уровни доступа

| Уровень       | Условие открытия                                               | Пул Q сырья        | Скидка | Срок поставки |
| ------------- | -------------------------------------------------------------- | ------------------ | ------ | ------------- |
| Рыночный      | всегда                                                         | 20–55 (равномерно) | 0%     | 1–3 дня       |
| По знакомству | 3+ успешные сделки с NPC-поставщиком                           | 35–75              | 5–15%  | 1–2 дня       |
| Эксклюзивный  | сословная реп. купцов ≥ 50 **и** отношение с поставщиком ≥ 0.7 | 60–95              | 10–25% | 0–1 день      |

### 3.2. Формула цены партии (заказа)

```
price = base_price × volume × quality_price_mult(Q) × (1 - discount) × (1 + urgency_fee)
```

| urgency_fee | Условие                          |
| ----------- | -------------------------------- |
| 0%          | обычный заказ                    |
| +15%        | срочно (−1 день к сроку)         |
| +30%        | сегодня (доставка в тот же день) |

### 3.3. Минимальные объёмы заказа (пилотные ресурсы)

| Ресурс | base_price (Q=50) | Мин. объём |
|--------|-------------------|------------|
| Бревно | 4 деньги | 10 шт. |
| Доска | 1.5 деньги | 20 шт. |
| Мех (шкура) | 6 денег | 5 шт. |
| Фурнитура (комплект) | 15 денег | 1 шт. |

### 3.4. Шаблон для новых ресурсов

Любой новый ресурс описывается полями: `id`, `base_price`, `min_volume`, `unit`, `decay_days` (опционально), `season_mod` (опционально).

## 4. Производство и мастерские

### 4.1. Уровни мастерской

| Уровень | Стоимость апгрейда | Слотов | workshop_bonus (к Q_out) | Содержание/день |
|---------|-------------------|--------|--------------------------|-----------------|
| 0 — Надворная | 0 | 1 | +0 | 1 деньга |
| 1 — Мастерская | 120 | 2 | +8 | 3 деньги |
| 2 — Цеховая | 450 | 4 | +15 | 8 денег |
| 3 — Мануфактура | 1500 | 8 | +22 | 20 денег |

### 4.2. Время производства

```
production_days = base_days × (100 / max(50, worker_efficiency + workshop_bonus))
```

- `worker_efficiency` — средняя эффективность нанятых на этом слоте (см. §5), типично 55–125.
- Минимум `production_days = ceil(base_days × 0.5)` — даже с мастерами нельзя ускорить больше чем в 2 раза.

### 4.3. Рецепты (формат данных)

```json
{
  "id": "saw_planks",
  "inputs": [{"resource": "log", "qty": 3}],
  "outputs": [{"resource": "plank", "qty": 10, "q_inherits": "formula"}],
  "base_days": 2,
  "skill_used": "woodworking"
}
```

## 5. Найм рабочих и подрядчиков

### 5.1. Рыночная зарплата

| Роль | Рыночная ЗП/день |
|------|------------------|
| Подмастерье / новичок | 2 деньги |
| Ремесленник | 3.5 деньги |
| Мастер | 5 денег |
| Караванщик / лоцман | 4 деньги |

### 5.2. ЗП vs квалификация

Игрок указывает ЗП в объявлении как **% от рыночной** для данной роли.

| ЗП (% рыночной) | Кто откликается | Эффективность | Ошибки/день | Кражи/месяц |
|-----------------|-----------------|---------------|-------------|-------------|
| < 70% | подмастерья, бродяги | 55–70 | 15% | 8% |
| 70–99% | обычные ремесленники | 75–90 | 8% | 3% |
| 100–130% | опытные | 95–110 | 4% | 1% |
| > 130% | мастера | 110–125 | 2% | 0.5% |

### 5.3. Процесс найма

- Объявление активно **7 дней**.
- Число откликов: `3 + floor(offered_wage / market_wage) + random(0..2)`.
- Игрок выбирает до `слотов мастерской` кандидатов.

### 5.4. Последствия ошибки и кражи

| Событие               | Эффект                                      |
| --------------------- | ------------------------------------------- |
| Ошибка (производство) | потеря 20–40% партии **или** Q_out −15..−25 |
| Кража                 | потеря 5–15% стоимости склада за месяц      |

## 6. Договоры и риски неграмотности

Связь с §2.6: `literacy` от 0.0 до 1.0.

detect_chance = literacy × 0.7 + (scribe_hired ? 0.25 : 0)

## 7. Продажа, рынок, конкуренция NPC

### 7.1. Цена продажи

```
sell_price = base_market_price × quality_sell_mult(Q_out) × (1 + reputation_premium)
reputation_premium = min(0.25, merchant_reputation / 200)
```

- `merchant_reputation` — сословная репутация среди купцов, 0..100 (может быть выше с бонусами).
- Игрок может выставить цену **±20%** от расчётной; отклонение > +20% снишает шанс продажи (см. §7.2).

### 7.2. Спрос на рынке

| Стадия игры | Слотов покупателей/день | Доля слотов NPC-конкурентов |
|-------------|-------------------------|-----------------------------|
| Смерд | 5 | 60% |
| Купец | 12 | 75% |
| Боярин | 20 | 80% |

**Шанс продажи в первый день:**

```
sale_chance = base 40%
  + 30% если Q_out > market_avg_Q
  + 15% если price ≤ calculated_sell_price
  − 10% за каждые 10% завышения цены сверх calculated
  + reputation_premium × 100 (в процентных пунктах, max +25)
```

### 7.3. Караван

| Параметр                               | Значение                                        |
| -------------------------------------- | ----------------------------------------------- |
| Длительность маршрута (пилот: Балтика) | 25 дней                                         |
| Множитель выручки к локальной продаже  | ×1.4 .. ×2.0 (зависит от Q и сезона)            |
| Риск события за день пути              | 12%                                             |
| Заморозка капитала                     | стоимость груза + аванс караванщику до возврата |

**События каравана (веса):**

| Событие | Вес | Эффект |
|---------|-----|--------|
| Задержка | 35% | +3..+7 дней |
| Грабёж | 25% | потеря 30–60% груза |
| Повреждение | 20% | Q_out −10..−20, цена −15% |
| Удачная сделка | 15% | выручка +20% |
| Без происшествий | 5% | — |

Аванс караванщику: **15%** от ожидаемой выручки при отправке.

## 8. Пилотная цепочка: «Лес → Ладья → Караван»

### 8.1. Схема

```
Бревно (Q_in) → [Распилка, 2 дня] → Доски (Q_mid)
Доски × 30 + Фурнитура → [Сборка ладьи, 14 дней] → Ладья (Q_out)
Ладья → Продажа в Новгороде  ИЛИ  Караван (25 дней)
```

### 8.2. Таблица этапов

| Этап | Вход | Выход | base_days | base_market_price (Q=50) |
|------|------|-------|-----------|--------------------------|
| Закуп бревна | — | 1 бревно | — | 4 деньги |
| Распилка | 3 бревна | 10 досок | 2 | — |
| Сборка ладьи | 30 досок + 1 фурнитура | 1 ладья | 14 | — |
| Продажа локально | 1 ладья | — | — | 180 денег |
| Караван | 1 ладья | — | 25 | 280–340 денег |

### 8.3. Расчёт Q для эталонного прогона

**Допущения:** Q бревна = 62, skill_craft = 55, мастерская ур. 1 (workshop_bonus = 8), 2 ремесленника по 100% ЗП (efficiency ≈ 85).

После распилки:
```
Q_planks ≈ min(62) × 0.4 + 55 × 0.35 + 8 × 0.25 = 24.8 + 19.25 + 2 = 46 → Обычное
```

После сборки ладьи (вход Q_planks = 46):
```
Q_boat ≈ 46 × 0.4 + 55 × 0.35 + 8 × 0.25 = 18.4 + 19.25 + 2 = 39.65 → Низкое/граница Обычного
```

Для целевого Q_boat ≈ 60 нужно Q бревна ≥ 75 или skill ≥ 70, либо мастерская ур. 2.

**Целевой прогон (Q_boat ≈ 60):** Q бревна = 72, skill = 60, мастерская ур. 1:
```
Q_boat ≈ 72×0.4 + 60×0.35 + 8×0.25 = 28.8 + 21 + 2 = 51.8
```
+ бонус мастера (efficiency 100%): +3 к округлению навыка → **Q_boat ≈ 58** (Обычное/Хорошее).

### 8.4. Экономика эталонного прогона (Q_boat ≈ 60)

| Статья                  | Расчёт                   | Сумма    |
| ----------------------- | ------------------------ | -------- |
| Бревна                  | 40 шт. × 4 × 1.35 (Q≈72) | 216      |
| Брак при распилке (~5%) | +2 бревна                | 11       |
| Фурнитура               | 1 комплект               | 15       |
| ЗП распилка             | 2 дня × 2 раб × 3.5      | 14       |
| ЗП сборка               | 14 дней × 2 раб × 3.5    | 98       |
| Содержание мастерской   | 16 дней × 3              | 48       |
| **Итого затрат**        |                          | **~402** |
| Продажа локально        | 180 × 1.25 (Q≈60)        | **225**  |
| **Маржа локально**      |                          | **−177** |

> **Вывод v0.1:** при текущих константах полный цикл «лес → ладья» **убыточен** при локальной продаже. Караван обязателен как цель цепочки.

| Караван | 300 × 1.25 | **375** |
| Аванс караванщику | −15% | −56 |
| **Чистая выручка каравана** | | **319** |
| **Маржа с караваном** | 319 − 402 | **−83** |

Цепочка всё ещё убыточна — нужна **оптимизация** (поставщик со скидкой 15%, своя заготовка леса, один мастер вместо двух ремесленников) или **корректировка base_price ладьи**.

### 8.5. Сбалансированный прогон v0.1 (целевая маржа 18%)

**Корректировки для прототипа:**
- `base_market_price` ладьи (Q=50): **180 → 240** денег локально.
- Закуп бревен «по знакомству» со скидкой 10%: 40 × 4 × 1.35 × 0.9 ≈ **194**.
- Один мастер (5 ден/день) + один подмастерье (2 ден): (2×3.5) + (14×5) + (14×2) = 7 + 70 + 28 = **105** ЗП.
- Содержание: 16 × 3 = **48**.

| Статья | Сумма |
|--------|-------|
| Сырьё + фурнитура | 209 |
| ЗП | 105 |
| Мастерская | 48 |
| **Итого** | **362** |
| Локальная продажа (Q=60) | 240 × 1.25 = **300** |
| Караван (ожидание) | 380 × 1.25 = **475** |
| Минус аванс 15% | −71 |
| **Чистый караван** | **404** |
| **Маржа каравана** | **+42 (~11.6%)** — ближе к цели; windfall/скидки доводят до 15–25% |
