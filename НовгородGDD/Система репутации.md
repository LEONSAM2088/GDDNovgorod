# Rules — правила изменения

1. [[Запомнить поступок (свидетель)]]
2. Слухи → копирование `Knowledge` с меньшей уверенностью.
3. Ассоциация (A с B на виду) → у D слабые теги A←B и B←A.
4. Смена статуса / публичное событие → обновление `Status` или публичных тегов.
5. Оценка: `RepOpinion = f(Traits(B), Knowledge(B→A))` → см. [[Формирование репутации]].
6. Сделка на виду: NPC соглашается, если выгода > **ожидаемая цена тегов** у свидетелей (цена = конфликт тега с **своими** трейтами). Без свидетелей цены почти нет.
7. [[Система репутации (Схема).canvas]]

# Dynamics — что возникает в игре из правил

- В одном круге «хороший кузнец», в другом «убийца холопа» — разные реакции.
- Слухи разъезжаются: часть города знает, часть нет.
- После публичной сделки с «плохим» растёт риск ярлыков у свидетелей.
- Игрок целенаправленно качает имя в нужной аудитории, а не «репутацию вообще».

# UI / Player-facing - что видит игрок

- тон и ветки при первой встрече («слышал о тебе…»)
- готовность говорить / торговать / в союз (до личного доверия)
- отказ контакта на виду (сделка с репутацией)
- разный приём в разных кругах
- (опц.) уровни словами, не числа

# Telemetry / Debug

**Минимальный срез:** A помогает купцу при свидетелях → у купца и круга тег «надёжен»; незнакомый купец с трейтом «ценит надёжность» — теплее.

**Логировать:** кто какой тег получил, conf, источник (видел / слух / ассоциация).

**Тест:** подстава слухом → тег только у части NPC, не у всех.
